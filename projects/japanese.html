日文道場 — Nihongo Dojo

一個以「讓自學者真的學得下去」為核心命題的 Side Project。
透過遊戲化留存機制、完整的學習閉環設計，加上 Material Design 3 沉浸式 UI，
從零到一由一人完成設計、開發與發布。

角色

背景：為什麼要做這個？

開始自學日文後，我快速發現一個矛盾：市面上的學習工具要嘛功能太複雜（讓人不知從何開始），要嘛太枯燥（背了三天就放棄）。
Duolingo 雖然好，但 N5 文法的課表與台灣人習慣的說法不完全契合。

這個問題對我而言是真實需求，也是一個可以同時練習「設計思維 + PM 產品規劃 + 前端實作」的好機會。 於是我決定：自己打造一個專屬的學習工具。

User Pain Point

自學者缺乏即時回饋，學完一個單元不知道自己「到底學進去沒有」，導致動力快速流失。

Market Gap

台灣繁體中文 N5 學習資源以靜態教材為主，互動性強且設計品質高的工具稀缺。

Opportunity

遊戲化 + 深度客製化內容（台灣情境的會話練習）+ 高品質 UI，三者同時到位的競品幾乎為零。

日文道場的差異化主軸是：台灣學習者視角 × 遊戲化留存 × 沉浸式 UI，三者合一。

功能架構：以留存為核心的產品設計

從 PM 視角出發，每個功能的存在都必須服務於「讓使用者明天還想回來」這個核心目標。 以下五大模組各自對應不同的學習階段與留存機制：

📚
課程系統

結構化的 N5 學習路徑，含單字、文法、會話三個維度，清楚顯示進度與解鎖狀態，製造「下一關」的驅動感。

🃏
單字卡（Flashcard）

3D 翻牌動效 + 語音朗讀（TTS）。「不記得 / 記得」雙選反饋，模擬間隔記憶邏輯，加速記憶鞏固。

🎯
測驗（Quiz）

四選一選擇題 + 填空輸入雙模式，答題後即時顯示正誤反饋與解釋，搭配愛心系統製造適度緊張感。

🔗
配對遊戲（Matching）

日文 ↔ 中文連連看，答錯時震動動畫 + 扣心機制，讓「失敗成本」成為記憶的錨點。

📖
文法說明

重點句型 + 波浪底線標注 + 多組例句，採台灣日常情境（購物、問路、餐廳），讓語法學習不脫離實用脈絡。

💬
情境會話

分角色的對話場景（A/B 雙角色 + 店員/客人），搭配 TTS 逐句朗讀，模擬真實對話節奏。

商業思維：遊戲化留存機制拆解

參考 Duolingo 的核心留存邏輯，並針對「沒有帳號系統的 Side Project」重新設計適合的輕量化版本：

🔥 連勝天數（Streak）

最強的回訪驅動力之一。使用者一旦建立「連勝記錄」，為了「不讓數字歸零」而每天回來的動機遠超過「想學習」本身。這是 Duolingo 留存率的核心支柱之一。

⚡ XP 升級系統

每完成一個練習獲得 XP，累積至 500 XP 升級。進度條讓使用者清楚看到「距離下一個里程碑還差多少」，製造 Endowed Progress Effect（已擁有進度捨不得放棄）。

❤️ 愛心（Lives）系統

測驗中限制犯錯次數，答錯扣心。適度的失敗成本讓使用者認真作答，避免無腦亂點，也讓「心全部消耗」成為一個自然的回訪觸發點。

✨ 即時正向反饋

每次答對顯示 XP 浮動數字動畫（+10 XP），答題完成後呈現「太棒了！」完成頁，搭配正確率、用時、獲得 XP 等統計，讓每次練習都有明確的成就感。

設計決策：為什麼選 Material Design 3？

Dark Mode First

深夜睡前記單字是高頻使用情境。深色介面降低眼睛負擔，也讓日文假名在深色背景上更易辨讀。主色 Cyan-Blue（#4FC3F7）在深色背景有高對比，符合 WCAG 可及性標準。

Adaptive Navigation

手機 → Bottom Navigation Bar；平板 → Nav Rail；桌機 → Navigation Drawer。一套 CSS + media query 實現三段響應式導航，與 MD3 官方設計規範完全對應。

Motion Design

單字卡翻牌、畫面切換、XP 浮動數字全部有專屬動畫。Emphasized Easing（cubic-bezier(.2,0,0,1.4)）讓互動有彈性感，提升操作的愉悅度。

零框架純實作

以原生 HTML / CSS / JS 完成。

反思：如果這是一個真實產品，接下來要做什麼？

以 PM 視角來看，目前的版本是一個功能完整的 MVP（Minimum Viable Product）。 如果要往「真正的產品」演進，我會優先做以下三件事：

  • 建立帳號系統 + 雲端進度同步：目前進度存在 localStorage，換裝置即消失。Streak 的留存力量必須跨裝置才能真正生效。
  • 導入分析儀表板：追蹤「哪些單字答錯率最高」「使用者在哪個步驟流失」，以數據驅動課程內容迭代，而非憑感覺新增教材。
  • 社群 / 排行榜機制：Duolingo 最強的留存手段之一是同儕競爭。加入「本週積分排行」或「揪朋友一起學」，可大幅提升網路效應與自然分享率。
PM Takeaway #1

遊戲化不是「加點數就好」，需要理解每個機制背後的心理學原理，才能設計出真正有效的留存工具。

PM Takeaway #2

「設計者即使用者」的 Side Project 雖然驗證成本最低，但也最容易陷入個人偏見；真正的 PM 工作需要超越自己的使用情境去思考。

PM Takeaway #3

從零到一獨力完成讓我深刻理解「設計可行性」：哪些互動效果在純 CSS 實現時有技術天花板，哪些需要 JS 支援，影響後續與工程師溝通的精準度。
